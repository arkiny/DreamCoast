> 물려** 머티리얼 해석 결과를 재사용한다.

## Context (왜 / 무엇)

**클러스터 기반 *virtual geometry*** 를 자체 엔진에 구현한다. 목표는 "수억 삼각형을 화면
픽셀 예산에 맞춰 GPU-driven으로 그린다"는 핵심 아이디어를 두 백엔드(D3D12·Vulkan)에서 직접
체험·학습하는 것. 학습 가치가 가장 큰 다섯 기둥을 모두 다룬다:

1. **클러스터 LOD DAG** — 메시를 meshlet(클러스터, ~128 tri)로 쪼개고, 클러스터를 그룹으로 묶어
   그룹 단위로 단순화 → 다시 클러스터로 분할. 부모/자식 간 **단조 증가 에러 경계**를 기록한 DAG.
   그룹 경계를 LOD마다 바꿔 **크랙(seam) 없이** 클러스터 단위 LOD를 섞는 것이 이 기법의 핵심 트릭.
2. **뷰 종속 컷 선택(cut selection)** — 런타임 컴퓨트가 DAG를 훑어 "자신의 화면 투영 에러는 OK,
   부모는 너무 큼"인 클러스터 집합(= cut)을 고른다. 팝핑/크랙 없는 연속 LOD.
3. **GPU 컬링 + 2-pass 오클루전(HZB)** — 절두체 / 백페이스(노멀 콘) / 소형 클러스터 컬링 후,
   지난 프레임 HZB로 1차 컬 → 렌더 → 새 HZB → 1차 탈락분 재검사(2차)로 false-negative 회수.
4. **소프트웨어 래스터라이저** — 픽셀 이하 삼각형은 HW 래스터가 비효율 → 컴퓨트 셰이더가
   직접 래스터, **64-bit 아토믹 max**로 `(depth<<32 | clusterTriId)`를 비저빌리티 버퍼에 기록.
   큰 클러스터는 **메시 셰이더(HW)** 경로로 분기(화면 크기 기준 binning).
5. **비저빌리티 버퍼 + 머티리얼 해석** — 래스터 단계는 셰이딩 없이 `(clusterId, triId)`만 기록.
   별도 resolve 패스가 픽셀별로 삼각형을 되짚어 속성 보간(해석적 barycentric) → **Phase 6 G-buffer**
   기록 → 기존 디퍼드 PBR 라이팅이 그대로 소비.

> 의도적으로 **풀 경로**(SW 래스터 + 비저빌리티 버퍼 포함). 메시 셰이더만 쓰는 단계적 경로가
> 아니라, 미세 삼각형 SW 래스터까지 목표. 다만 마일스톤을 쪼개 HW 경로를 먼저 세운 뒤 SW를 얹는다.

## 확정 사항 (사용자)

- **풀 경로**: 클러스터 LOD DAG + GPU 컬링/HZB + **컴퓨트 SW 래스터 + 비저빌리티 버퍼** + 머티리얼 해석.
- 로드맵상 **신규 Phase 10**(Phase 7 컴퓨트 전제, Phase 6 디퍼드와 통합). 실험적 트랙으로 진행.

## 신규 RHI / 인프라 요구사항 (블로커 후보)

Phase 6/7에서 들어오는 것 외에, 이 Phase가 RHI에 처음 요구하는 것들:

| 기능 | Vulkan | D3D12 | 비고 |
|---|---|---|---|
| 메시 셰이더 파이프라인 | `VK_EXT_mesh_shader`(task+mesh) | Mesh Shader (SM6.5) | RTX 2070 SUPER(Turing) 지원 |
| 64-bit 버퍼 아토믹 | `VK_KHR_shader_atomic_int64` (`shaderBufferInt64Atomics`) | SM6.6 64-bit atomics | 비저빌리티 버퍼 `atomicMax` |
| 64-bit 이미지 아토믹(대안) | `VK_EXT_shader_image_atomic_int64` | (R32G32 또는 SM6.6) | 가능하면 **R64 스토리지 버퍼**로 통일 |
| 인다이렉트 카운트 드로우/디스패치 | `drawMeshTasksIndirectCountEXT` / `dispatchIndirect` | `ExecuteIndirect` + count | GPU-driven 제출 |
| 버퍼 디바이스 어드레스 | `VK_KHR_buffer_device_address` | 기본(GPU VA) | 클러스터 페이지 포인터 체이싱 |
| 컴퓨트 mip 다운샘플 | 공통 | 공통 | HZB(max) 생성 |

- **이식성 리스크(최우선 검증)**: 64-bit 아토믹 모델이 두 API에서 가장 갈린다. 1차 목표는
  **R64(uint64) 스토리지 버퍼 + `InterlockedMax`/`atomicMax`** 단일 경로. D3D12는 SM6.6 64-bit
  raw-buffer 아토믹, Vulkan은 `shaderBufferInt64Atomics`. 둘 중 하나라도 막히면 **R32 depth +
  별도 R32 payload, 2-단계 락리스 비교** 폴백을 설계한다. → **M0 게이트에서 캡쳐로 증명.**
- 메시 셰이더 확장은 `VK_EXT_mesh_shader`(표준) 우선, 없으면 `VK_NV_mesh_shader` 폴백 여부는 M0에서 결정.

## 의존 외부 크레이트 (사용자 승인 필요 — minimal-install 원칙)

오프라인 빌더는 검증된 라이브러리에 기대는 편이 학습/안정성 모두 유리:

- **`meshopt`**(meshoptimizer FFI) — `build_meshlets`, `simplify`(에러 반환), `optimize_*`. 사실상 표준.
- **그래프 파티셔너** — 그룹핑(클러스터→그룹)에 METIS류 필요. `metis` 크레이트(FFI) 또는
  경량 자체 구현(공유 엣지 기반 greedy) 중 택1. **M1 착수 시 사용자와 확정.**
- 나머지는 기존 스택(`glam`, `gltf`, 자체 RHI)으로 충분.

> 외부 의존은 `meshopt`(+선택적 `metis`)로 최소화. 자체 구현 대체안도 각 마일스톤에 병기.

## 데이터 모델 (클러스터 페이지)

빌드 산출물 = **클러스터 페이지 스트림**. 클러스터 1개:

- 로컬 정점 버퍼 인덱스 범위, 로컬 인덱스(8-bit, ~128 tri), 정점 수.
- **바운딩 스피어**(컬링/에러 투영), **노멀 콘**(백페이스 클러스터 컬), 머티리얼 ID.
- **LOD 에러**(이 클러스터를 그리는 비용)와 **부모 그룹 에러**(부모로 올라갈 때 경계) — 단조성 보장.
- 그룹 인덱스, 부모/자식 링크(DAG; 트리 아님 — 자식 여럿이 부모 여럿과 그룹 공유 가능).

런타임 버퍼: 상주 클러스터 풀(SoA), 인스턴스 테이블, 가시 클러스터 리스트(append),
HW/SW 분기 리스트 2개, 인다이렉트 인자 버퍼, HZB 피라미드, 비저빌리티 버퍼(R64).

## 마일스톤

### M0 — RHI 케이퍼빌리티 게이트 (블로커)
- 메시 셰이더 파이프라인(`task`+`mesh` 스테이지) RHI 추가, 64-bit 버퍼 아토믹, 인다이렉트 카운트
  드로우/디스패치, BDA 노출. 각 기능 **케이퍼빌리티 감지 + 피처 플래그**(없으면 명확히 실패).
- 최소 검증: 메시 셰이더로 하드코딩 삼각형 1개, 컴퓨트에서 R64 버퍼 `atomicMax` 왕복.
- **게이트**: 두 백엔드에서 위 두 스모크가 검증 레이어 클린. 64-bit 아토믹 경로 RenderDoc/PIX 확인.

### M1 — 오프라인 클러스터 빌더 (asset, CPU)
- `meshopt`로 meshlet 생성(~128 tri) → 그룹핑(~8–32 클러스터/그룹) → **그룹 단위 단순화**(공유
  경계 엣지 lock, 삼각형 절반) → 재분할 → 부모 LOD. 재귀로 루트(단일/소수 클러스터)까지.
- 각 단계 에러를 **부모≥자식 단조**로 보정해 DAG 에러 경계 기록. 클러스터 페이지로 직렬화.
- 디버그 뷰어(기존 메시 경로 재사용): 클러스터/LOD/그룹 컬러링, 통계(클러스터 수·LOD 레벨·에러 분포).
- **게이트**: 샘플 메시가 N개 LOD로 빌드, 직렬화→로드 라운드트립, LOD별 시각화에 **크랙 없음**.

### M2 — 상주 메시 셰이더 렌더 (HW 경로, 컬링 無)
- 모든 클러스터 GPU 업로드. 고정 LOD를 **메시 셰이더로 래스터**(task=클러스터 디스패치,
  mesh=정점/프리미티브 방출). 기존 디퍼드 G-buffer 셰이딩으로 일단 직결(비저빌리티 버퍼 전 단계).
- **게이트**: 클러스터 컬러/실제 머티리얼 두 모드로 정상 렌더, 두 백엔드 일치.

### M3 — LOD DAG 컷 선택 (compute, 뷰 종속)
- 컴퓨트가 DAG를 훑어 **화면 투영 에러 임계값** 기준 cut 선택("self error ≤ τ ∧ parent error > τ").
- 선택 클러스터 → append 리스트 → **인다이렉트 메시 드로우**. 거리/FOV에 따른 연속 LOD.
- **게이트**: 카메라 이동 시 팝핑/크랙 없이 디테일 전환, 삼각형 수 화면 적응. PIX/RenderDoc로 cut 확인.

### M4 — GPU 컬링 + HZB 2-pass 오클루전
- per-클러스터 절두체 + 노멀 콘 백페이스 + 소형(서브픽셀 바운드) 컬링. 인다이렉트 카운트로 압축.
- **HZB**(컴퓨트 max 다운샘플) 생성. **2-pass**: ①지난 프레임 HZB로 컬 후 렌더+새 HZB →
  ②1차 탈락분을 새 HZB로 재검사해 false-negative 렌더.
- **게이트**: 가려진 클러스터 드로우 수 급감(통계), 폐색 경계에서 누락/깜빡임 없음.

### M5 — 비저빌리티 버퍼 + 소프트웨어 래스터라이저
- 화면 크기 추정으로 클러스터를 **HW(메시 셰이더) / SW(컴퓨트)** 두 리스트로 binning.
- SW 경로: 컴퓨트가 삼각형 래스터, **R64 `atomicMax`**로 `packed = (depth<<32)|(clusterId<<7|triId)`
  기록. HW 경로도 같은 비저빌리티 버퍼에 기록(픽셀 셰이더/`SV_Barycentrics`).
- **게이트**: 미세 삼각형 밀집 메시가 SW 경로로 정확 렌더(z-fighting/누수 없음), HW/SW 경계 seam 없음.

### M6 — 머티리얼 해석 → G-buffer (Phase 6 통합)
- resolve 패스: 비저빌리티 버퍼에서 `(clusterId, triId)` → 삼각형 정점 페치 → **해석적
  barycentric**(스크린 도함수)로 속성 보간 → Phase 6 G-buffer(albedo/normal/material) 기록.
- 이후 파이프라인은 기존 디퍼드 라이팅(직접광/섀도우/IBL/톤매핑) 그대로.
- 머티리얼 다중 시: 머티리얼별 타일/클래시파이 후 resolve(초기엔 단일 풀스크린 컴퓨트로 단순화).
- **게이트**: 비저빌리티→G-buffer→PBR 최종 이미지가 M2 직결 경로와 시각 일치, 두 백엔드 동일.

### M7 — (스트레치) 스트리밍 / 페이지 상주
- 클러스터 페이지 단위 on-demand 상주: GPU 피드백(요청 버퍼) → CPU 업로드 → 폴백(부모 LOD) 상주.
- 초기 범위 밖. 메모리 한계 도달 전까지 **전량 상주**로 진행, 인터페이스만 페이지 친화적으로 설계.

### M8 — 툴링 & 마무리
- ImGui 디버그: 클러스터/LOD/그룹/HZB/비저빌리티/HW·SW 분기 시각화, 컬링·LOD·SW 토글, 통계 HUD.
- 본 문서 한계/검증 채우고 ROADMAP Phase 10 상태 갱신.

## 검증 전략
- `cargo fmt --all` + `cargo clippy --workspace --all-targets -- -D warnings` clean.
- `cargo run -p sandbox -- --backend vulkan|d3d12` 양쪽 안정 렌더, **두 백엔드 시각 일치**.
- Vulkan 검증 클린(`VK_LOADER_LAYERS_DISABLE=~implicit~`). RenderDoc/PIX로 비저빌리티 버퍼·HZB·
  cut·64-bit 아토믹 경로 캡쳐 검증.
- 회귀 메시: (a) 고밀도(미세 삼각형) 스캔/조각상, (b) 큰 평면(HW 경로), (c) 폐색 헤비 씬(오클루전 검증).

## 알려진 리스크 / 미결
- **64-bit 아토믹 이식성**(D3D12 SM6.6 ↔ Vulkan 확장) — M0에서 단일 경로 확정, 폴백 설계.
- **메시 셰이더 확장 차이**(`VK_EXT_` vs `VK_NV_`) — M0에서 결정.
- **그룹핑 파티셔너**(METIS 의존 vs 자체 greedy) — M1 착수 시 확정.
- DAG 에러 단조성/크랙-프리 보장 — M1·M3에서 시각·수치 검증.
- SW/HW binning 임계값 튜닝, 머티리얼 해석의 다중 머티리얼 분류 비용 — M5·M6 측정 후 조정.
- 스트리밍(M7)은 스트레치. 초기엔 전량 상주.
