# 실시간 GI 충실도 로드맵 — 상용급 SW-RT GI 도달

상위: [ROADMAP.md](ROADMAP.md). 이 문서는 **실시간 GI/라이팅의 "충실도(구조) 갭"** 을 상용 게임 엔진
수준으로 좁히는 계획이다. 성능(같은 화질을 더 적은 비용으로)은 별도 트랙([sw-rt 성능 트랙](swrt-gi-perf-track.md))이,
게임 런타임/툴 breadth는 [갭 분석](commercial-engine-gap-analysis.md)이 다룬다. **이 문서는 "얼마나
정확·견고·확장 가능한가"** 만 본다.

## 0. 성공 기준 (무엇을 "도달"로 보나)

레퍼런스 상용 엔진의 *출력과 비트 일치*가 목표가 아니다(불가능·무의미). 우리의 성공 척도는 셋:

1. **물리적 정확도 = 자체 path-tracer 잔차** (ground truth). 각 개선은 콘텐츠에서 PT 잔차를 낮추거나 최소 중립.
2. **견고성 = 드롭·구멍 없음.** 지오메트리/라이트가 예산·정렬 문제로 라이팅에서 누락되지 않을 것.
3. **확장성 = 큰 씬에서도 성립.** 메모리/추적 비용이 씬 규모에 비례해 폭주하지 않을 것.

불변 게이트(전 페이즈): **갤러리 바이트 동일**(앵커 `af70c1a5…`), **DX≡VK ≤0.001**(Windows 해제 시),
**결정론**(run-to-run 바이트 동일), **단일소스**, **heavy=opt-in / RenderQuality 티어**, **상표명 금지**.

## 1. 이미 닫은 충실도 갭 (기반)

- **per-mesh SDF 직접 샘플(하이브리드 coarse+atlas)** — 얇은 지오 관통·표면캐시 카드 오등록 해결
  ([per-mesh-sdf-direct-sample-plan.md](per-mesh-sdf-direct-sample-plan.md)). 표면캐시 뷰 dense 38%→direct 100% 채움.
- **하늘 시드 occluded skylight 게더** — 간접 게더 miss에 절차 하늘(occluded), 옛 unoccluded ambient 제거.
- **스크린 라디언스 프로브 GI** + **SH-L1 방향 이라디언스 프로브 + 실내 skylight occlusion**.
- **cone-trace LOD march / sparse GI 프로브**(성능 트랙) — 추적 원점·step 감소.

## 2. 남은 구조적 갭 (진단 — 코드 근거)

| 시스템 | 현재 (파일/상수) | 상용 기대 (일반 기법) | 충실도 영향 |
|---|---|---|---|
| 표면 캐시 할당 | 6면 고정 카드/draw, `MAX_CARDS=1024`, ~170 draw 수용, 초과분 **카드 없음** (`fuse.rs`) | 데맨드 기반 가상 카드(페이지 풀·LRU·mip), 카메라/중요도 레지던시 | **최대 갭** — 큰 씬에서 지오가 캐시에서 누락(라이팅 구멍) |
| mesh SDF 저장 | dense `dim³` 타일, 캡 32³, nearest-union, 단일 셀그리드 (`sdf_atlas.rs`) — sparse/brick/mip **전무** | 좁은 밴드만 저장하는 스파스 브릭 + cone LOD용 mip + 메시 컬링 | 메모리·해상도·메시 수 상한 → 확장성 제약 |
| 추적 경로 | SW sphere-march 전용; HW-RT는 PT(ground truth) 전용, 실시간 GI 미사용 (`gi.rs`/`reflect.rs`) | HW-RT 충실도 경로(hit 라이팅, march 오차 0)를 High 티어로 | 원거리·고주파에서 march 오차·누설 |
| 라디언스 캐시 | 월드 캐시 1레벨 + 스크린 프로브 + **코사인** 게더 | 다중 클립 월드 라디언스 캐시 + **중요도 샘플** 파이널 게더 | 원거리 GI 노이즈·수렴 속도 |
| in-cache 재질 | albedo는 dense 그리드, emissive/양면 미반영 | per-mesh albedo, emissive 방출, 양면(식생) | 색 정밀도·발광체·식생 투과 |
| 다중 바운스 | 1바운스 게더 + 이전 프레임 캐시 피드백(시간적 다중바운스) | 지속 캐시 통한 물리 다중바운스 + radiosity | 실내 밝기·수렴(중간 우선순위) |

## 3. 페이즈 로드맵 (충실도 영향 × 방금 깐 토대 활용 순)

각 페이즈: 착수 전 `docs/phase-*` 승인 → 검증된 단일 커밋. 부분 병렬 가능.

### F1 — 표면 캐시 가상화 (최우선; 관측된 최대 갭)
**왜:** sponza_intel에서 448 draw 중 278이 카드 예산 초과로 **드롭**(라이팅 누락). 큰 씬일수록 악화.
**목표:** 하드 드롭 제거 — 모든(가시) 지오가 캐시에 참여.
**접근:**
- (a) **데맨드 레지던시** — 카드 요청 피드백(카메라 프러스텀 + GI/반사가 실제 샘플한 draw)으로 필요한 카드만 상주.
- (b) **페이지 아틀라스 + LRU 방출** — 고정 슬롯 대신 페이지 풀; 예산 초과 시 최근 미사용 카드 방출·재사용(드롭이 아니라 스트리밍).
- (c) **카드 mip** — 원거리 draw는 저해상 카드로(비용↓, cone 게더와 정합).
**게이트:** PT 잔차(콘텐츠 정성) 개선/중립, 갤러리 바이트 동일(콘텐츠 전용 seam), 결정론, DX≡VK.
**리스크:** 재조명이 GI/반사 소비자 공유 → 회귀 검증 필수. 페이지 방출의 결정성(시드 고정).

### F2 — mesh SDF 확장성 ✅ (실측 재스코프 랜딩: 비등방 dims + f16; 브릭 보류)
**결과(2026-07-13):** S1 실측이 브릭 전제를 반박(점유 74–85% → apron ×1.5 손해) → 재스코프.
아틀라스 71.0→**28.6 MB(×0.40)**, per-axis 캡 48, PT budget PASS(interior −0.20 개선). 브릭은
점유 <40% 씬에서 재평가, mip 체인은 별도 페이즈([phase-f2-mesh-sdf-scalability-plan.md](phase-f2-mesh-sdf-scalability-plan.md)). 아래 원문은 이력.

### (원문) F2 — 스파스 브릭 mesh SDF (F0 아틀라스의 진화)
**왜:** dense 타일은 대부분 far-from-surface 복셀에 메모리 낭비(캡으로 완화했지만 근본은 dense). 확장성 상한.
**목표:** 좁은 밴드만 저장 → 메모리↓·해상도↑·더 많은 유니크 메시; cone 트레이싱용 mip.
**접근:**
- occupied **브릭(예: 8³)만** 아틀라스에 팩 + 인디렉션 볼륨(브릭 좌표→아틀라스 페이지). 빈 브릭은 coarse 폴백.
- 브릭 **mip 체인**으로 cone LOD 샘플(F1 카드 mip·cone-trace와 정합).
- 이게 표면캐시 카드 캡처 정밀도도 동반 상승(등가면 더 정확).
**게이트:** 직접샘플 대비 PT 잔차 중립+메모리 감소 보고, 갤러리 불변, DX≡VK. 결정론적 브릭 팩.
**의존:** 방금 shipped된 per-mesh 아틀라스/인스턴스/셀그리드 인프라 재사용.

### F3 — HW-RT 충실도 경로 (High 티어; SW/HW 분리)
**왜:** SW march는 스케일용이나 원거리·고주파에서 오차·누설. 레퍼런스는 HW-RT를 충실도 경로로 둠. 우리 HW-RT는
현재 ground-truth PT에만 존재.
**목표:** 동일 `cm_geo_*` / 게더 seam 뒤에 **HW-RT 트레이스 백엔드**를 High 티어로. SW=디폴트(스케일), HW=충실도.
**접근:** BLAS/TLAS(이미 rt.rs 보유) 재사용, 게더/반사 히트를 HW-RT로, hit에서 표면캐시/라디언스 캐시 라이팅 읽기.
**게이트:** High 티어 opt-in(디폴트는 SW — 성능 트랙 원칙 유지), **PT 잔차 SW보다 개선**, DXR≡VK_KHR 파리티, 결정론.
**리스크:** 백엔드 파리티(DXR vs VK_KHR), Metal RT 경로. 별도 트랙으로 격리.

### F4 — 계층적 월드 라디언스 캐시 + 중요도 파이널 게더
**왜:** 원거리 GI가 코사인 게더라 노이즈·수렴 느림; 월드 캐시 단일 레벨.
**목표:** 다중 클립 월드 라디언스 캐시(원거리 저비용 조회) + 중요도 샘플 파이널 게더(같은 spp에 노이즈↓).
**접근:** 카메라-추종 클립 레벨의 라디언스 프로브 계층 + BRDF/조도 기반 중요도 방향 샘플. 스크린 프로브와 결합.
**게이트:** 콘텐츠 PT 잔차 개선, 정적 셔머(CAPTURE_SEQ) 무회귀, DX≡VK, 결정론.

### F5 — in-cache 재질 충실도
**왜:** 색 정밀도(albedo dense), 발광체·양면 식생 미반영.
**목표:** (a) **per-mesh albedo 아틀라스**(F0 SDF 아틀라스와 동형; 히트 색 정밀), (b) **emissive** 재질을
캐시 방출로(발광체가 GI에 기여), (c) **양면/투과 식생**을 캐시 캡처·게더에서 처리.
**게이트:** 각 항목 PT 잔차 개선/중립, 갤러리 불변.

### F6 — 검증·견고성 인프라 (가로지름)
- **콘텐츠 PT 잔차 자동화** — ✅ Part A(PT 자동노출) + Part B(lit-마스크 잔차 게이트, `golden-image.py`
  `sponza_pt_*` config — [phase-f6b-content-pt-residual-plan.md](phase-f6b-content-pt-residual-plan.md)).
- **DX≡VK 해제 시** 전 F 페이즈 Windows 재검증(현재 동결·보류).
- **골든이미지 회귀 CI**(양 백엔드 매트릭스, 결정적 시드) — 표면캐시/GI 셔머 회귀 자동 포착.

## 4. 의존 / 권장 순서

```
F1 표면캐시 가상화 ─┐
F2 스파스 브릭 SDF ─┼─(토대; 서로 보강: 브릭 정밀↑ → 카드 캡처 정밀↑)
                    │
F3 HW-RT 충실도(High) ── 병렬 가능(별도 파리티 트랙)
F4 계층 라디언스 캐시 + 중요도 게더 ── F1/F2 후
F5 in-cache 재질 ── F1 후(카드 인프라 위)
F6 검증 인프라 ── 연속(각 F가 자기 몫)
```
**1차 수직 슬라이스(가장 큰 충실도 이득/공수):** `F1 → F2 → F4`. 이 셋이면 "큰 씬에서 드롭 없이, 스파스
고해상 필드로, 저노이즈 다중바운스 GI"가 선다. F3(HW High)·F5(재질)는 이후 폭 확장.

## 4b. F6 심화 아크 상태 (2026-07-17 봉인 — phase-f6i-sky-chain-recalib-plan.md §0 필독)

E-오라클→F6E~F6K 13유닛의 감사·수리 아크 완결. **출하 스택은 interior 28.115→26.541로
개선·봉인**(block64 게이트, 하향 래칫). 실험 심(P_SDF_OPEN_UNSIGNED)은 근본 물리 회복
(bias≈0)에도 **이진 SDF의 결정론적 팬텀 얼룩으로 시각·게이트 열세 — 기본 OFF 봉인**.
이진-부호·해상도·시간-평균 계열은 **전부 실측 기각(재시도 금지 — §0 원장)**. 재개 경로는
확률적-점유 필드 표현(근본)뿐이며 새 phase-doc 승인부터.

**F6L (2026-07-20, phase-f6l-trapped-probe-fill-plan.md)**: 히어로 블랙-박스 수리 —
진범 = **bent-normal 정규화의 신뢰도 파괴**(V≈0 셀의 밴드-1 노이즈 → 풀-강도 랜덤
bent). 수리 = **유효도 계약**: 프로듀서가 방향×유효도(`sat(√3·c0/|b|)` — SH 비음성
경계, 무튜닝·스케일-불변) 기록, 소비자는 |bent|=신뢰도로 bent-프리 계약에 페이드.
게이트 sunlit 19.159/interior 26.599 PASS·갤러리 비트-동일·sc_viz 재시드. 잔여
직교-개구 암전은 배너 knob `P_SKYVIS_BENT_FLOOR=0.25`(레시피 전용)로, 완전 해소는
확률적-점유 트랙. 부수 실결함 2건(all-rejected 하드-블랙 소비 폴백, 갇힌/내장 셀 주입
충전)은 `P_GI_TRAPPED_FILL` knob(기본 OFF — sunlit +0.23, 자체 캘리브 패스 대기).

## 5. 하지 말 것
- PT 잔차 없이 "정확/도달" 단정. 갤러리 앵커 무단 변경(라이팅 개선이면 PT 검증 후 재기준선).
- HW-RT를 **디폴트** 경로로(파리티 리스크 — High 티어 opt-in). SW가 스케일 디폴트.
- 단일소스 위반(각 소비자가 따로 필드/캐시 접근). 상표명 산출물 사용. heavy 기본 ON 강제.
- 측정 없는 최적화(성능 트랙 1원칙 유지).

## 6. 현실 점검 (정직)
표면캐시 가상화·스파스 브릭·HW-RT 충실도·계층 라디언스 캐시는 각각 **수 주~수개월** 규모의 R&D다. 전체는
다년 과업일 수 있다. 이 문서의 가치는 "전부 완성"이 아니라 **(a) 충실도 갭의 정직한 우선순위, (b) 방금 깐
per-mesh 인프라를 어디에 재사용하는지, (c) 각 페이즈를 PT-잔차로 검증 가능한 단위로 자르는 구조**를 박아두는
데 있다. 각 F는 기존 방식대로 착수 전 `docs/phase-*` 승인 → 검증된 단일 커밋으로 랜딩.
