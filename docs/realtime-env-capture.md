# Real-time environment capture (camera-based) — plan

> Phase 6 IBL은 시작 시 1회 큐브 캡처. 이 문서는 **실시간(매 프레임/타임슬라이스) 환경 캡처**로
> 확장하는 계획. 모두 표준 split-sum IBL / 실시간 리플렉션 프로브 기법이며 컴퓨트 없이
> 그래픽스 패스로 구현 가능.

## 실전 성능 기법 (표준 split-sum IBL)
1. **캡처 → 밉 체인 → 밉에서 컨볼루션.** 캡처는 cube mip0에 렌더, 다운샘플로 mip 체인 생성,
   스페큘러 프리필터는 **소스 밉 체인**을 샘플.
2. **프리필터 32~64 샘플**(`roughness<0.1 ? 32 : 64`). 샘플마다 소스 밉을
   `mip = 0.5*log2(solidAngleSample / solidAngleTexel)`로 선택 → 적은 샘플 + 파이어플라이 없음.
   `solidAngleSample = 1/(numSamples*pdf)`, `pdf = D_GGX·0.25`,
   `solidAngleTexel = 4π/(6·size²)·2`. roughness<0.01 → mip0 복사(미러).
3. **roughness ↔ mip 매핑**: `mip = maxMip - (1 - 1.2·log2(roughness))` 류의 단조 매핑.
4. **디퓨즈 irradiance**: 정석은 SH 9계수(컴퓨트). 컴퓨트 없으면 저해상도 irradiance 큐브
   유지 또는 env 상위 밉 근사.
5. **타임 슬라이싱 + 더블 버퍼**: 캡처/컨볼루션을 여러 프레임에 분산, convolved 큐브 2개 핑퐁
   (메인 패스는 항상 완성된 쪽 샘플). 무거운 패스는 면당 1프레임 / 저해상도로 분산.
6. 스페큘러 컨볼루션은 **픽셀 셰이더(그래픽스) 경로**로 충분 → 컴퓨트 불필요.

## 우리 엔진 적용 (그래픽스 전용)
- **env 큐브에 밉 체인 추가**: 캡처는 mip0, 다운샘플 패스로 mip 1..N 채움(절차적 스카이는
  방향 함수라 각 밉을 직접 렌더 가능 → 다운샘플 자기참조 해저드 회피).
- **prefilter 재작성(완료, RT1)**: 32~64 샘플 + 밉 기반 샘플. roughness↔mip 매핑.
- **파이프라인화**: `submit_oneshot`+`fence.wait` 제거 → 캡처/컨볼루션을 프레임 커맨드버퍼에
  메인 그래프 **앞**에 기록(스톨 없음).
- **카메라 위치 씬 캡처(RT3)**: 면별 90° 포워드 패스(깊이)로 sky+지오메트리를 mip0에 렌더,
  **직접광만**(IBL 미적용 → 재귀 없음). 이때 실제 다운샘플(서브리소스 배리어) 필요.
- **더블 버퍼 + 타임 슬라이싱**: 매 프레임 전체 재생성이 비싸면 도입(저해상도 + 32~64 spp면
  매 프레임도 가능할 수 있으니 먼저 측정).

## 마일스톤 (게이트: 빌드+clippy+양 백엔드+Vulkan 검증+스크린샷+FPS 확인)
- **RT1 — 밉 체인 + 저비용 프리필터(성능 토대) ✅:** env 밉 체인 + 전 밉 스카이 렌더, prefilter를
  32~64-샘플 밉 기반으로. IBL 품질 동등↑ & 비용↓ 확인(양 백엔드).
- **RT2 — 인루프 파이프라인 재생성 ✅:** 캡처+컨볼루션을 프레임 루프로(`submit_oneshot`+wait 제거,
  메인 그래프 앞에 기록). irradiance 16²/저샘플+blurred-mip로 경량화. UI "Real-time env capture"
  토글(off 시 태양 변경에만 재캡처). **~1.17ms/frame**(약 850 FPS) 측정.
- **RT3 — 카메라 위치 씬 캡처 ✅:** 포워드 `capture.slang`로 **지면 평면**을 env mip0에 카메라
  위치 기준 렌더(단일 쿼드 → 깊이 불필요, 반사 물체 제외 → 자기 반사 회피). 금속 표면이 지면을
  반사(하단), 양 백엔드 픽셀 일치. 면별 90° 뷰프로젝션 + Vulkan flip로 면 정렬.
- **RT4 — 마무리 ✅:** UI 토글, `MeshPosNormal` 정점 레이아웃(검증 클린), 문서/메모리. 배경
  스카이 분리는 **불필요 확인**(지면은 env 하반구에만, 배경은 상반구→하늘). FPS/검증 클린.
- **RT5 — 멀티바운스(더블 버퍼) ✅:** env/irradiance/prefilter 큐브를 **2벌 핑퐁**. 매 프레임
  씬을 write 세트에 캡처할 때, 그 표면을 **read 세트(이전 프레임)의 IBL**(split-sum:
  irradiance·albedo + prefilter·(F·LUT.x+LUT.y))로 셰이딩 → 반사 물체가 반사 안에서도 반사로
  보임(멀티바운스). 캡처는 항상 *다른* 세트를 읽으므로 재귀/스톨 없음, 수 프레임에 걸쳐 수렴.
  메인 라이팅은 항상 갓 쓴 write 세트 샘플. 패리티는 **실제 캡처 시에만** 전진(스킵 프레임은
  마지막 write 세트 유지 → 깜빡임 없음). 루프 진입 전 두 세트를 단일바운스로 1회 초기화(첫
  프레임이 미초기화 메모리 샘플 방지). `capture.slang` push 112→208B(+model/eye/metallic/
  roughness/이전 IBL 인덱스; RTX 2070 maxPushConstants 256 이내). UI "Multi-bounce
  reflections" 토글(off=이전 단일바운스 단일 큐브 동작). 검증: 양 백엔드 픽셀 일치, Vulkan
  검증 클린, on/off 픽셀 diff가 크롬 구 영역에 국한(2차 반사라 거친 표면은 거의 불변).

## 알려진 한계 / 차후
- 캡처 씬 = 지면 평면만(자기 반사 회피). 모델 등 볼록 지오메트리를 넣으려면 **깊이 버퍼 또는
  백페이스 컬링** 캡처 파이프라인 필요 — 다음 단계.
- 큐브가 카메라 위치 기준 → 떨어진 물체 반사에 시차 오차(표준 리플렉션 프로브). 추후 parallax 보정.
- 디퓨즈 irradiance를 SH로(컴퓨트 도입 시). 현재는 저해상도 큐브.
- env mip0만 지면 포함, mip 1..N은 하늘만 → 거친 반사는 지면 기여 적음(샤프 반사는 정확).
