> 상태: **B안 채택·게이트 랜딩** (2026-07-16, 사용자 승인) — block64_avg가 주 게이트
> (예산 interior 28.41 / sunlit 19.2 시드, 하향 래칫 유지), masked_avg·bias·scatter·
> block16/32·dark는 섀도. 후속 = fine·hardvis 재심 + tint 재캘리브(§4b 하단). 동기 =
> E-오라클 페이즈
> ([phase-e-oracle-plan.md](phase-e-oracle-plan.md) §2f)가 세 번째로 같은 벽에 부딪힘:
> **E-도메인(물리)에서 레퍼런스에 근접하는 수리가 디스플레이 게이트에서 벌점**을 받는
> bias→scatter 전환 문제. 게이트 정책 변경은 이전 페이즈들에서 명시적 비목표였으므로
> (F6D·interior-bias §4), 본 문서는 근거를 사전 등록하고 **정책 선택을 사용자 결정으로
> 올리는 논의 문서**다. 결정 전에는 어떤 게이트 코드도 변경하지 않는다.

## 1. 현행 게이트의 구조 (사실 확인)

- `rt-compare.py`: lit-마스크(PT luma > 8) 픽셀의 채널별 |raster−PT| 평균 = **masked_avg
  (게이트)**. F6D가 추가한 분해: masked_bias(부호 있는 평균 = 평탄 오프셋),
  masked_scatter(평균에 대한 MAD = 공간 산포) — **섀도 전용**(기록만, 게이트 없음).
  |bias| ≤ masked_avg ≤ |bias| + scatter.
- `golden-image.py`: masked_avg ≤ residual_budget, budget은 하향 래칫(--update가
  measured + 0.3 마진으로 재봉인).
- 현행 봉인(2026-07-16, `fe6dce9`): sunlit 27.684 = bias +5.18 / scatter 27.98,
  interior 31.608 = bias +16.92 / scatter 28.96.

**핵심 관찰: 양 게이트 모두 avg의 대부분이 ~28의 scatter 페데스탈이다.** bias가 0이 되어도
avg ≈ 28 — 게이트 여유의 실질 전부가 산포 페데스탈에 잠식되어, 남은 개선 레버가 bias 축소
하나뿐이다. 그리고 bias 축소의 가장 값싼 수단은 **평탄 필**(tint류)이다.

## 2. 문제 사전 등록 — 축적된 차단 사례 (전부 기록 완료 수치)

| 사례 | 물리(E-도메인/분해) 방향 | 게이트 판정 | 출처 |
|---|---|---|---|
| ① fine 기본 ON | lit 비율 1.078→1.008(bias 실질 소거), F6D 분해 bias −5.2/−7.5 | scatter +0.81/+2.19 → **avg 악화, 차단** | F4B `ce65469`·F6D `ff077e6` |
| ② E-수리 풀스택 + t0.1 | interior bias **+18.13→+8.82(−9.3)**, E-도메인 PT-방향 확정 | avg 31.61→32.56(+0.95 악화 — scatter +3.7이 잠식) → **FAIL** | E-oracle §2f |
| ③ `P_GI_SUN_HARDVIS` | E-도메인 lit 36.4→40.7(PT 48.8) — 태양-체인 결손의 직접 수리 | t0.2 이중충전 FAIL / t0.1 sunlit 30.8 폭발 — **tint 창 공집합, knob 강등** | E-oracle §2f |

구조적 원인: masked_avg는 (a) **표현-스케일보다 미세한 구조 불일치에 가혹**하고(32³ 프로브
≈ 1.1m 격자가 PT의 픽셀-구조를 정렬할 수 없음 — 구조를 추가하면 정렬 실패분이 전부 산포로
과금) (b) **저주파 오배분에 관대**하다(평탄 필은 lit 평균만 맞추면 무벌점). 즉 현행 게이트는
"평탄하고 평균만 맞는" 근사를 "구조적으로 옳지만 미세-정렬이 어긋난" 근사보다 선호한다 —
build-to-quality 디렉티브(PT 충실도)와 반대 방향의 선택압.

**사각지대(별건이지만 같은 결정에 묶임)**: lit-마스크는 pt_black(인테리어 36%)을 제외한다.
tint 0.2의 암부 2.24× 과충전(interior-bias 페이즈 EV11 실측)은 게이트에 직접 보이지 않고
AE 미터링을 통해 간접 투영만 된다 — interior-bias 페이즈의 부호-반전 혼란의 근원.

## 3. 후보 정책

### A. 분해 게이트 (bias 주 게이트 + scatter 회귀 가드)
masked_avg 예산을 폐지하고: `|masked_bias| ≤ bias_budget`(하향 래칫, 현행 값 시드) +
`masked_scatter ≤ scatter_budget`(하향 래칫이되 **1회 착지 마진** — 정답-방향 트레이드의
착지 공간). masked_avg는 섀도로 강등(기록 유지).
- 장점: 도구 무변경(이미 계산·기록됨), 러너 게이트 로직만 수정. 해석 명료.
- 한계: **평탄-필 선호는 그대로다**(평탄 필은 bias도 최소화). 사례 ②는 scatter 마진을 +4
  줘야 재개방 — 그 폭이면 회귀 가드가 물러진다. 사례 ①은 마진 +1.5로 재개방 가능.
- 부호 상쇄 리스크: bias는 +영역과 −영역이 상쇄될 수 있음 — scatter 가드가 방어하지만,
  bias-단독 개선 커밋의 시각 리뷰를 규약에 추가해야 함.

### B. 블록-평균 잔차 (표현-스케일 정합 메트릭) — 판별 측정 겸용
라스터·PT를 N×N 픽셀 블록 평균으로 다운샘플 후 lit-블록 |diff| 평균. N은 프로브 투영
크기에서 유도해 사전 등록(interior 카메라에서 1.1m ≈ 48-64px). **같은 틀에서 암부 트랙**:
pt_black 블록의 raster 평균 luma ≤ dark_budget — 평탄 필의 암부 과충전을 직접 게이트
(현행 유일 사각지대 폐쇄).
- 장점: "복셀-스케일 misalignment 용서 + 저주파 오배분 과금" — 문제 정의와 정확히 쌍대.
  **판별력**: 사례 ②·③의 scatter +3.7이 고주파 정렬 실패라면 블록-평균에서 소멸하고,
  진짜 저주파 오배분이라면 잔존한다 — 채택 여부와 무관하게 이 측정 자체가 hardvis의
  재개방 가부를 판정한다.
- 단점: rt-compare 확장 필요(블록 축적), N 자유도(사전 등록으로 고정), 새 예산 시드 필요.

### C. E-도메인 보조 게이트 (GI-트랙 전용)
EV11 고정노출 뷰15 vs PT E-뷰(crop luma·lit-mean·B−R) — AE 디커플, GI 필드의 물리 직접
측정. E-oracle 페이즈에서 도구·레시피·기준값 전부 확립됨.
- 장점: GI 추정자 변경에 최강 판별력(이번 페이즈가 실증). AE-투영 혼란 원천 차단.
- 한계: GI-외 변경(반사·섀도·재질)에 무감 — 기존 게이트 **대체 불가**, GI 페이즈 규약의
  보조 앵커로만.

### D. 지각 메트릭(SSIM 등) — **기각 제안**
감사 가능성(단일 숫자의 물리 해석)이 낮고 도구 스택이 무거움. 프로젝트의 "단순·감사 가능한
메트릭 + 래칫" 스타일과 충돌.

## 4. 권고안 (단계적)

1. **1단계 — B의 판별 측정을 먼저** (게이트 무변경, rt-compare에 섀도 블록 지표만 추가):
   사례 ②·③ 구성의 공식 쌍을 재캡처해 블록-평균 잔차를 계산. **hardvis의 scatter가
   표현-스케일 정렬 실패(블록에서 소멸)인지 진짜 오배분(잔존)인지**가 정책 선택의 최대
   불확정 요소이므로, 이 측정이 A/B 사이 결정을 데이터로 바꾼다.
2. **2단계 — 정책 랜딩** (측정 결과에 따라):
   - 블록에서 소멸 → **B 채택**(블록-평균 lit 게이트 + 암부 트랙, masked_avg·bias·scatter
     전부 섀도 강등) → hardvis·fine 재심 → tint 재캘리브 근거 확보.
   - 블록에서 잔존 → **A 채택**(bias 주 게이트 + scatter 가드, 착지 마진 +1.5) — hardvis는
     정당하게 차단 유지, fine은 재개방 심사.
   - 어느 쪽이든 **C를 GI 페이즈 규약에 편입**(러너 행 승격은 선택, 최소한 수동 앵커 규약).
3. 예산 시드·래칫 규칙·시각 리뷰 규약은 채택안 확정 후 본 문서에 추가 사전 등록.

판독:
- **§3B의 가설 실증**: 게이트가 기각한 두 구성(hardvis+t0.1·fine)의 scatter 초과분은
  64px-이하 구조 정렬 실패였고 블록 평균에서 증발한다(i_hv_t01: per-pixel 32.6 →
  block64 27.5). 진짜 저주파 오배분(hardvis t0.2 이중충전)은 블록에서도 최악 —
  판별력 보존. N=16에서도 역전은 이미 발생(fine 29.70 < 기본 30.31), N=64에서 명확.
- **정직 노트**: sunlit hardvis는 블록에서도 +0.47 열세 + 암부 luma 상승 — hardvis는
  자체 캘리브 이벤트(B-게이트 하 tint 재스윕) 없이 자동 통과가 아니다. 암부 트랙은
  AE-매개라 절대 게이트로는 혼탁(fine의 E-암부 개선이 AE 재정규화로 상쇄된 65.2가 증거)
  — 섀도 유지, 암부의 절대 가드는 C(EV11 E-도메인 행)의 몫.
- 결정론: 블록 지표는 같은 캡처 이미지의 파생이므로 캡처 결정론(AE FIXED_DT,
  masked_avg 소수 3자리 재현 실적)을 그대로 상속.

**갱신된 권고안**: §4의 분기 조건이 "블록에서 소멸"로 판명 → **B 채택**을 제안:
block64_avg를 주 게이트로(예산 시드 = 현행 기본 측정값 interior 28.115 / sunlit 18.901
+ 마진, 하향 래칫 유지), masked_avg·bias·scatter·block16/32·dark는 섀도 강등,
C(EV11 E-행)는 GI 페이즈 규약 편입. 채택 시 즉시 후속: fine·hardvis 재심(블록-게이트
개선 실측 이미 확보)과 tint 재캘리브. **게이트 코드 변경은 본 제안의 승인 후.**
