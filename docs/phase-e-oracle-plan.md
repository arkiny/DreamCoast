# E-오라클 계획서 — 볼륨 라디언스 E의 ~8× 에너지 결손: 업데이트측 회계 감사

> 상태: **진행 중 — 계획** (2026-07-15). 동기 =
> [phase-interior-bias-root-plan.md](phase-interior-bias-root-plan.md) §5d — interior lit
> 결손 −17.6/255의 운반자여야 할 볼륨 E 항이 실측 ~2.2/255(≈1/8). 이 결손이 tint 크러치
> (=디스플레이 bias +18.13)와 fine 산포 문제의 공통 상류다. 스칼라·형상·스코프 레버는 전부
> 소진(F4B2·F4B3·interior-bias 3중 확인) — 남은 것은 **E 추정자 체인의 단계별 감사**뿐.

## 0. 질문 (사전 등록)

볼륨 업데이트(gi_volume.slang)의 E 추정 체인에서 PT 대비 에너지가 어디서 새는가:
① 히트 셰이딩(태양 항: `gv_shadow` 32스텝 소프트 섀도·`sun.w·ndl/π`) ② `sky_fill`×V(B2)
③ 멀티바운스 피드백(`alb×indirect` — 감쇠·절단) ④ 마치 자체(48스텝·`maxt` 클램프·bias 0.05 —
히트 놓침=미스=0) ⑤ albedo 소스(`cm_albedo` 0.7 폴백·`GROUND_ALBEDO`) ⑥ SH-L1 투영/재구성
(클램프드-코사인 A0/A1 — 각도 대역 손실) ⑦ 소비측(occ-가중 재정규화·트라이리니어).

## 1. 오라클 설계 — PT E-뷰 (스크린-공간, 기존 계측 인프라 재사용)

**정의 정합이 핵심**: 볼륨 E는 "장면-바운스 전용" 규약(미스=0, 하늘은 IBL이 별도 공급 —
gi_volume.slang:349-357). 오라클은 같은 규약의 PT측 대응물이어야 한다.

- **PT E-뷰 모드**(`P8_PT_EVIEW=1`, rt.rs PT 셰이더의 디버그 모드): 1차 카메라 히트에서
  코사인-가중 반구 적분 — 반구 레이가 지오메트리를 히트하면 그 지점의 **풀 PT 셰이딩 라디언스**
  (연속 경로 포함), 하늘로 이탈하면 **0**. 출력 = E_PT(n) (albedo 곱하기 전, 노출 전 —
  DEBUG_VIEW=10과 같은 물리량).
- **raster E-뷰** = 기존 `DEBUG_VIEW=10`(gdf_gi 볼륨 E(n)) — 무수정.
- 비교: 고정 EV11, interior 카메라, rt-compare(+F6D bias/scatter) + crop-luma — E-도메인에서
  직접 결손 지도를 얻는다. 갤러리/게이트 무접촉(디버그 모드 전용).
- 주의: PT E-뷰의 수렴(반구 spp × 프레임 누적)은 PT 관례(WARMUP=192·8spp) 재사용; 미스=0
  규약 때문에 pt_black이 커지므로 lit-마스크는 raster E-뷰 luma 기준으로 뒤집어 적용(도구
  플래그 재사용 검토 — 필요시 rt-compare에 `--lit-mask-src=left` 추가, 기존 계약 불변).

## 2. 증분

1. **오라클 랜딩**: PT E-뷰 모드 + 검증(맨해튼 새니티: E_PT가 lit 콜로네이드에서 대략
   17.6/255 스케일의 결손 좌변을 재현하는지 — interior-bias §5b와 정합해야 오라클이 옳다).
   갤러리·게이트 불변(디버그 플래그 전용), 단일 커밋.
2. **감사 행렬**(env/셰이더 프로브, 커밋 없음): §0의 각 단계를 볼륨측에서 토글/완화하며
   E_raster가 E_PT에 접근하는 정도를 측정 — 예: gv_shadow 스텝/상한(기각 이력의 sh≡1 재실험
   금지 — 명압 보존 완화만), 마치 96스텝, spp 64, albedo 상수 스윕, 멀티바운스 항 분리 뷰.
   각 행 = EV11 E-뷰 쌍 크롭/bias 비교(게이트-레시피 아님 — AE 무관 도메인).
3. **수리 증분**(행렬 판정 후 설계): 근본 단계의 수리를 opt-in seam으로 랜딩 → 공식 게이트
   배터리(budget 쌍 — interior 개선 기대 + bias 하향, sunlit 무회귀, 셔머, 갤러리, 풀
   매니페스트). 성공 시 tint 재캘리브(하향)는 후속 커밋(F6C 방법론 — E 회복이 근거 제공).

결함일 수 있다(공유 프리미티브라면 볼륨 업데이트의 프로브 마치도 같은 히트율 문제를 공유).

## 3. 게이트·규약

측정은 EV11 E-도메인(오라클) + 공식 러너(수리 증분만). 기존 규약 전부 승계: 같은-설정 쌍,
셰이더 빌드마다 "Compiling dreamcoast-shader" 확인(**pull 후 포함** — sc_viz 사건), cd 명시,
caffeinate, 상표명 금지, 계획서 커밋, 검증된 단일 커밋.

## 4. 비목표·반복 금지

PT 레퍼런스 본체 수정 / sh≡1(gv_shadow 제거 — interior 57 폭발 기각) / WRC 부활 /
spp-위장 비용 증가(수리는 정확도-중립 비용 우선, 비용 증가는 실측 트레이드 명시) /
tint 재스윕 단독(E 회복 근거 없이).
