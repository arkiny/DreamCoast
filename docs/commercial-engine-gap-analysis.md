# 상용 엔진급 확장 — 갭 분석 & 런타임/툴링 로드맵

상위: [ROADMAP.md](ROADMAP.md). 이 문서는 **렌더링 코어 이후** DreamCoast를 *범용 상용 게임 엔진*
수준으로 끌어올리기 위해 무엇을 더 추가해야 하는지의 **전략 검토 + 신규 Phase 계획**이다. 개별 Phase의
세부는 진행하며 `docs/phase-NN-*.md`로 분기한다.

## 1. 현 위치 평가 (정직한 진단)

**강점 — 렌더링/GPU 코어는 이미 상용 R&D급.**
- 3 백엔드 raw RHI(Vulkan/D3D12/Metal), 바인드리스, 렌더그래프, 디퍼드 PBR+IBL+섀도우, 컴퓨트/GPGPU,
  HW 레이트레이싱(+Ground-Truth 패스트레이서), 버추얼 지오메트리(P10), 레퍼런스식 SW-RT GI/반사
  (P11), GPU 컬링, async compute, GPU 프로파일러, RenderQuality 티어.
- 진행/계획: 쿡 에셋(P12), 자체 ECS+씬그래프+레벨 스트리밍(P13), 스켈레탈 애니메이션+GPU 스키닝(P14).

**공백 — "그릴 수 있다"는 많지만 "게임을 만든다"의 런타임·툴 계층이 거의 비어 있다.**
- 런타임 서비스: **잡 시스템(멀티스레드)·피직스·오디오·고급 입력·메모리/시간 관리** 부재.
- 게임플레이: **스크립팅·게임 UI·VFX 저작·고급 애니메이션·AI/내비·세이브/프리팹·네트워킹** 부재.
- 렌더링 완성도: **AA(TAA) 전무**, 투명/OIT, 다광원(클러스터드), CSM, 데칼, 대기/볼류메트릭, 지형/식생/
  물, 머티리얼 그래프, 버추얼 텍스처링 부재.
- 툴/프로덕션: **독립 에디터·핫리로드·CPU 프로파일러·메모리 추적·골든이미지 테스트·패키징·로컬라이제이션**
  부재.

> **요지:** 그래픽스는 깊은데 폭이 좁고, 게임 런타임/툴은 폭과 깊이 모두 비어 있다. 상용 breadth 목표는
> 이 **두 계층(런타임 서비스 + 저작 툴)**을 채우는 일이다.

## 2. 사용자 전략 결정 (이번 세션)

1. **구축 철학 = 하이브리드 + 인터페이스 분리.** 그래픽스/RHI는 from-scratch 유지. 피직스·오디오·스크립트·
   네트워킹 등 공수 크고 그래픽스 학습가치 낮은 시스템은 **성숙 라이브러리 통합**하되, **RHI처럼 자체
   facade trait 뒤에 백엔드를 격리** → 차후 from-scratch 자체 구현으로 *교체 가능*하게 설계(통합은 지금,
   자체화는 열어둠).
2. **목표 수준 = 범용 엔진 breadth.** 레퍼런스 엔진/Unity式 폭넓은 서브시스템 패리티 지향(최대 범위).
3. **에디터 = 독립 에디터 앱.** 본격 standalone 에디터(콘텐츠 브라우저·도킹·기즈모·인스펙터·undo/redo·
   PIE). 샌드박스 ImGui 툴링을 넘어선다.

## 3. 추가 설계 원칙 (기존 5원칙에 보강)

**6. 비그래픽스 대형 시스템은 "RHI 패턴"으로 통합.** 각 시스템마다 **from-scratch facade 크레이트**
(중립 타입 + trait)와 **백엔드 크레이트**(서드파티 어댑터)를 분리 — `rhi`/`rhi-vulkan` 구조를 그대로
미러링. 예: `crates/physics`(facade) ↔ `crates/physics-rapier`(백엔드). 엔진 코드는 facade에만 의존 →
백엔드 교체(또는 자체 구현 `physics-dreamcoast`)가 엔진 무변경. 서드파티 바이너리/소스는 미커밋 +
`tools/` fetch 스크립트(minimal-install 방침, ufbx/slang/vulkan-layers와 동일).

**7. 잡 시스템이 멀티스레드의 척추.** 렌더그래프가 GPU 패스의 척추이듯, **work-stealing 잡 시스템**이
CPU 병렬의 척추다. ECS 시스템 병렬화, 병렬 RHI 커맨드 기록, 비동기 에셋 스트리밍, 피직스 스텝이 모두
그 위에 얹힌다 → **신규 런타임 계층의 P0 토대**(아래 Phase 15).

**8. 결정성·재현성 우선.** 멀티스레드/피직스/네트워크 도입은 프레임 **고정 타임스텝 sim 루프**와
결정적 실행(잡 스케줄 시드, 피직스 결정성 옵션)을 전제로 — 헤드리스 골든이미지 회귀와 양 백엔드 검증
정책을 깨지 않게.

## 4. 갭 분석 (시스템별 — 현 상태 · 상용 기대 · 권고)

`B`=integrate(백엔드 trait 뒤), `S`=from-scratch, `S/B`=facade는 자체·백엔드 통합.

## 5. 신규 Phase 로드맵 (트랙별 · 우선순위 · 의존)

기존 Phase 0–14 + Metal 트랙 이후. 트랙은 부분 병렬이나, **잡 시스템(Phase 15)이 다수의 토대**다.
티어: **T0** 토대 · **T1** 게임 런타임 코어 · **T2** breadth 확장 · **연속** 가로지르는 인프라.

### 토대 (T0)
- **Phase 15 — 잡 시스템 / 멀티스레드 코어 + 병렬 렌더 [S].** work-stealing 스케줄러, 태스크 그래프,
  병렬 RHI 커맨드 리스트 기록, 고정 타임스텝 sim 루프, 결정적 스케줄. *모든 멀티스레드 시스템의 전제.*
  신규 `crates/jobs`. 완료: ECS/렌더/스트리밍이 N코어로 스케일, 양 백엔드·검증·골든이미지 무회귀.

### 게임 런타임 코어 (T1)
- **Phase 16 — 피직스 [S/B].** `crates/physics`(facade: 콜라이더/리지드바디/캐릭터컨트롤러/쿼리/트리거
  중립 타입) + `crates/physics-rapier`(백엔드). 고정 dt 스텝, ECS 동기화(트랜스폼), 레이/쉐이프 캐스트.
  의존 15·P13. 완료: 강체/캐릭터/트리거가 결정적 스텝으로 동작, facade 교체 가능 증명(목 백엔드).
- **Phase 17 — 오디오 [S/B].** `crates/audio`(facade: 사운드/소스/리스너/버스/DSP) + `crates/audio-kira`
  (백엔드). 3D 공간화, 믹싱, 스트리밍, 어테뉴에이션. `tools/fetch-*` 미커밋. 의존 15. 완료: 3D 사운드+
  믹스 버스, facade로 백엔드 격리.
- **Phase 18 — 입력 & 플랫폼 서비스 [S].** 액션 매핑(리바인딩), 게임패드/다중 디바이스, 윈도/디스플레이/
  DPI, 설정·환경 영속화(serde). 의존 platform. 완료: 액션 기반 입력 + 게임패드 + 리바인딩 저장.
- **Phase 19 — ECS 성숙 + 프리팹/세이브 [S].** 시스템 스케줄링(잡 위), 이벤트/메시지, 변경 감지, 프리팹
  합성, 게임 상태 직렬화. 의존 15·P13. 완료: 병렬 시스템 스케줄, 프리팹 인스턴스화, 세이브/로드 왕복.

### 렌더링 완성도 (T1→T2)
- **Phase 20 — AA + 포스트 스택 + 투명 [S].** TAA(P14 모션벡터), 오토 노출·DoF·모션블러·컬러그레이딩,

## 6. 에디터 트랙 — `apps/editor` (독립 앱, 연속 진행)

샌드박스(ImGui in-app 툴링)와 **별개의 standalone 에디터 애플리케이션**. 엔진을 라이브러리로 링크하고
런타임 씬/에셋을 저작한다. 거대·연속이라 단일 Phase가 아닌 자체 트랙으로, 런타임 기능이 성숙하는 만큼
함께 자란다. 마일스톤:
- **E1 — 셸:** 도킹 레이아웃, 뷰포트(엔진 렌더 임베드), 패널 골격, 커맨드/undo-redo 스택.
- **E2 — 씬 편집:** 계층 트리, 선택, 트랜스폼 기즈모, 프로퍼티 인스펙터(ECS 컴포넌트 리플렉션), `.level`
  로드/세이브(P13).
- **E3 — 콘텐츠:** 에셋 브라우저, 임포트 파이프라인 UI(glTF/FBX→`.dcasset` 쿡, P12/P14), 썸네일.
- **E4 — 서브에디터:** 머티리얼 그래프(P24), 애니메이션/스테이트머신(P27), 파티클/VFX(P28), 지형/식생
  페인팅(P23).
- **E5 — 이터레이션:** Play-In-Editor(PIE), 라이브 링크(러닝 게임에 핫 에디트), 빌드/쿡/패키지 UI(P12).
- **전제:** P13(ECS/리플렉션) 위에 E1–E2 착수 가능; 이후 각 런타임 Phase가 대응 서브에디터를 동반.

## 7. 프로덕션 인프라 (가로지르는 연속 트랙)

특정 Phase가 아니라 전 Phase에 스며드는 항목 — 각 기능 Phase가 자기 몫을 함께 낸다.
- **핫리로드:** 셰이더(P12 M4 계획) → 에셋 → 스크립트(P25) → (가능 시) 코드.
- **프로파일링:** GPU(완료) + **CPU 트레이싱(스팬)**·프레임 캡처·스탯 HUD·메모리 추적/예산.
- **진단:** 구조적 로깅(보유) + 크래시 리포트 + 텔레메트리.
- **테스트:** `tools/rt-compare.py` → **골든이미지 회귀 CI**(양 백엔드 매트릭스, 결정적 시드).
- **패키징/배포:** P12 쿡 확장 → 패키지/플랫폼 타깃(콘솔/모바일은 후속).
- **로컬라이제이션:** 문자열 테이블·폰트 폴백(P26와 결속).

## 8. 권장 순서 / 현실 점검

**의존 그래프 핵심:** `15(잡)` → {피직스16·ECS19·멀티스레드 의존 전반} ; `P14(모션벡터)` → `20(TAA)` ;
`P13(ECS/리플렉션)` → {프리팹19·스크립트25·에디터 E2} ; `P7` → {다광원21·VFX28} ; `P12` →
{월드 스트리밍23·머티리얼24·패키징}.

**권장 1차 수직 슬라이스(breadth지만 플레이 가능한 게임까지 가장 빠른 경로):**
`15 잡 → 16 피직스 → 18 입력 → 19 ECS/세이브 → 20 AA/포스트 → 17 오디오 → 25 스크립팅 → 에디터 E1–E2`.
이 8개면 "움직이고·부딪히고·소리나고·스크립트로 로직 짜고·에디터로 배치하는" 최소 게임 루프가 선다.
이후 21–24(렌더 breadth), 26–30(UI/애님/VFX/AI/넷), 에디터 E3–E5로 폭을 확장.

**현실 점검(정직하게):** 레퍼런스 엔진/Unity 수준의 **breadth + 독립 에디터**는 통상 대규모 팀의 다년 과업이다.
하이브리드(통합) 전략이 공수를 크게 줄이지만, 그래도 이 로드맵 전체는 **수년 규모**다. 본 문서의 가치는
"전부 완성"이 아니라 **(a) 올바른 의존 순서, (b) 백엔드 추상화 seam(차후 자체화 가능), (c) 언제든
플레이 가능한 수직 슬라이스로 자를 수 있는 구조**를 박아두는 데 있다. 각 Phase는 기존 방식대로 착수 전
`docs/phase-NN-*.md` 승인 → 검증된 단일 커밋으로 랜딩.

## 9. 리스크 / 미결
- **서드파티 의존 승인(minimal-install):** Rapier/Jolt, kira/miniaudio, mlua(Luau)/wasmtime, Recast 등은 각 Phase
  착수 시 사용자 승인 — facade 뒤 격리 + `tools/` fetch로 방침 유지.
- **결정성 vs 멀티스레드/피직스:** 골든이미지·양 백엔드 회귀를 깨지 않으려면 고정 타임스텝 + 결정적
  스케줄/피직스 옵션 필요(Phase 15·8원칙).
- **에디터 ↔ 런타임 결합:** 에디터가 엔진을 라이브러리로 링크 → 리플렉션/직렬화(P13·19)가 선결. 순환
  의존 회피 위해 에디터는 항상 *소비자*.
- **범위 폭주:** breadth 목표라도 항상 "수직 슬라이스로 절단 가능" 유지(§8) — 폭을 넓히다 플레이 불가
  상태로 빠지지 않게.
- **백엔드 추상화 누수:** 피직스/오디오 등은 라이브러리 개념이 facade로 새기 쉽다 — 중립 타입을 보수적으로
  설계하고, 자체 구현 목 백엔드를 한 번씩 끼워 누수 검출(RHI가 D3D12/VK 양쪽으로 검증하듯).
