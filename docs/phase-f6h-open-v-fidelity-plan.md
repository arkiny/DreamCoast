# F6H 계획서 — 개방부 V-충실도: 스카이-가시성 필드의 개방 영역 과소평가 조사

> 상태: **진행 중 — 계획** (2026-07-17). 동기 =
> [phase-f6g-tint-recalib-plan.md](phase-f6g-tint-recalib-plan.md) §2 — tint 하향 시 sunlit
> lit bias가 −6.8로 함몰(t0.10에서 block64 20.841 vs 예산 19.2): 개방 아트리움의 래스터
> 스카이라이트가 leak 없인 부족하다. 가설 = **V 필드(SH-L1 스카이-가시성)가 개방 영역의
> 하늘 열림을 과소평가**(후보 기전: 비대화 필드의 과다 히트·SH-L1 각도 평탄화·소비측
> saturate 체인·프로브 간격). V가 회복되면 sunlit 플로어가 내려가 t≈0.10(색 앵커 정합,
> interior 26.196)이 착지한다.

## 1. 오라클 — PT V-뷰 (E-뷰 인프라 재사용)

`P8_PT_VVIEW=1`(flip_y bit3): 1차 히트에서 코사인 반구 1방향(E-뷰와 공유), **bounce-1
미스(하늘 이탈) = 1 / 히트 = 0**으로 종결 — 볼륨의 투영 규약(vis = hit ? 0 : 1,
gi_volume.slang)과 정확히 같은 양의 레이-트레이스 그라운드 트루스. 누적 평균 =
코사인-가중 V(n). 비교측 = 기존 `DEBUG_VIEW=13`(소비자 V — 스카이라이트가 실제로 쓰는
재구성 값). 양쪽 모두 같은 톤맵을 통과(뷰 13은 노출 미적용, PT는 EXPOSURE=1 행)하므로
직접 비교 가능. 디버그 전용 — 게이트·앵커 경로 무접촉.

## 2. 증분

1. **오라클 랜딩**: bit3 + rt.rs env — 풀 배터리 비트-동일 증명, 단일 커밋.
2. **감사 행렬**(커밋 없음): sunlit·interior 카메라에서 (뷰13, PT V-뷰) 쌍 → 크롭/전면
   평균과 공간 분포로 개방부 결손 정량. 결손이 크면 기전 프로브: ① 프로브 마치의 과다
   히트(비대화 — eps/스텝 완화 A/B) ② 소진→vis 1(현행)이 개방부에선 어느 방향인지
   ③ SH-L1 재구성의 상향 클램프/평탄화(E/E_iso식 대역 바운드) ④ 소비측(gv_level_occ·
   saturate·bent-normal 경로) 격리.
3. **수리 증분**: 고립 기전의 수리를 knob 심으로 → 공식 배터리 → **성공 시 F6G 재개방**
   (tint 하향 재심 — sunlit 플로어 실측 후 t 재캘리브·budget 하향).

## 3. 게이트·비목표

게이트: block64 예산(19.2/26.84, 하향만)·셔머·갤러리·풀 매니페스트·검증된 단일 커밋.
비목표: PT 본체 수정 / V의 스칼라 부스트(무근거 곱셈 레버 — 기전 수리만) / tint 동시
변경(수리 검증 후 F6G 절차로).
